# Attention Stack 注意力栈

> **先让人动起来，再谈 DFS 哲学。**  
> **跑偏不是失败，回来就行。**

## 产品理念

现有的任务管理工具本质上都是「被动记录」——它们帮你记住要做什么，但不帮你真正去做。

**Attention Stack** 的核心理念是：将注意力从「广度优先」(BFS) 引导为「深度优先」(DFS)。

- 🎯 **多栈 + 聚光灯模型**：可以有多个目标，但同时只能聚焦一个
- 📍 **DFS 深入**：选定一条路，扎下去，直到触底或明确此路不通
- ⏰ **主动拉回**：定时提醒「你还在做这个吗？」
- 🎮 **游戏化奖励**：小目标 + 高频奖励 = 正反馈飞轮

## 核心特性

### 双轨入栈
- **懒人模式**：一句话即可开始（≤10秒）
- **完整模式**：填写目标、交付物、预计时间

### 栈操作
| 操作 | 含义 |
|------|------|
| 深入 | 拆分成更小的子任务 |
| 完成 | 弹出栈顶，回到上层 |
| 放弃 | 标记为此路不通 |
| 挂起 | 保存状态，稍后继续 |
| 切换 | 显式切换到其他任务 |

### 奖励系统（恢复型）
- 🪙 **金币**：完成任务获得，深度奖励有上限（前3层有效）
- 🔥 **连击**：跑偏时「冻结」而非清零，完成一次即解冻
- ⭐ **等级**：经验值累积升级，展示长期成长

## 设计原则

1. **先让人动起来，再谈哲学** — 创建任务 ≤10秒
2. **奖励是为了拉回，不是上瘾** — 惩罚可恢复、奖励有上限
3. **跑偏是常态，不是失败** — 温柔唤醒，回来就行
4. **工具是脚手架，不是拐杖** — 引导用户自我负责
